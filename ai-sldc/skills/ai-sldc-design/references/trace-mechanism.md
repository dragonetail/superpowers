# 追溯机制

## 概述

追溯机制确保每个成果物都能追溯到其上游来源，形成完整的知识流动链条。

## 追溯关系类型

| 关系类型 | 说明 | 示例 |
|---------|------|------|
| derives_from | 派生自 | API设计 派生自 用户故事 |
| validates | 验证 | 测试用例 验证 用户故事 |
| implements | 实现 | 代码 实现 API设计 |
| measures | 度量 | 价值报告 度量 业务目标 |

## 追溯矩阵

追溯矩阵记录所有成果物之间的追溯关系：

```markdown
| 成果物ID | 成果物名称 | 上游追溯 | 下游被追溯 | 状态 |
|----------|-----------|----------|-----------|------|
| PRJ-001 | 业务目标/KPI | - | BIZ-004, RUN-004 | 已批准 |
| BIZ-004 | 用户故事 | PRJ-001 | ARC-004, TST-001 | 已批准 |
| ARC-004 | API设计 | BIZ-004 | ENG-001, TST-001 | 已批准 |
| TST-001 | 测试用例 | BIZ-004, ARC-004 | - | 进行中 |
```

## 变更影响分析

当某个成果物发生变更时，通过追溯矩阵快速识别受影响的下游成果物：

```
示例：业务目标(PRJ-001)变更
↓ 影响分析
├── BIZ-001 业务流程模型 (需评估)
├── BIZ-004 用户故事 (需评估)
│   ├── ARC-004 API设计 (可能受影响)
│   │   ├── ENG-001 代码 (可能受影响)
│   │   └── TST-001 测试用例 (可能受影响)
│   └── TST-001 测试用例 (需评估)
└── RUN-004 价值度量 (需更新度量指标)
```

## 追溯覆盖率计算

```
追溯覆盖率 = 有追溯的成果物数 / 成果物总数 × 100%

目标：每个阶段追溯覆盖率 ≥ 80%
```

## 追溯维护规则

1. **新建成果物** - 必须填写 `traces_to` 字段
2. **修改成果物** - 评估下游影响，通知相关方
3. **删除成果物** - 检查是否被下游追溯，评估影响
4. **版本更新** - 保留历史追溯关系

## 元数据中的追溯表示

```yaml
---
id: ARC-004
name: API设计
traces_to:
  - BIZ-004    # 追溯到用户故事
  - BIZ-002    # 追溯到业务对象模型
---
```
