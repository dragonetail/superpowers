## 一、概述

### 1.1 背景与目标

GTMC数字化SLDC（软件生命周期开发）成果物体系旨在建立AI时代的数字化知识资产体系，实现：

- **业务驱动工程**：基于业务知识驱动的软件工程（Business-Driven Engineering）
- **AI协作适配**：让AI能够理解、生成、复用企业数字化知识
- **知识流动闭环**：【业务价值定义 → 需求要件整理 → 技术方案设计 → 代码实现交付 → 系统运行验证 → 价值度量反馈】

### 1.2 核心价值

| 价值维度 | 说明 |
|---------|------|
| 标准化输出 | 每次过程产出生成符合规范的文档 |
| 阶段间衔接 | 各阶段成果物之间有清晰的追溯和关联 |
| 质量把关 | 每个阶段的成果物有检查清单/评审标准 |
| 知识沉淀 | 将经验固化成模板，支持知识复用 |

### 1.3 参考标准

本体系参考以下行业标准和框架：

- Gartner 软件工程体系
- 数字化转型最佳实践
- AI-Coding 技术规范

## 二、行业对齐与视角分析

### 2.1 行业标准参考

| 标准/框架 | 核心视角 | 可借鉴点 |
|-----------|---------|---------|
| **ISO/IEC 12207** | 系统生命周期过程 | 过程分类：协议、组织、项目、技术 |
| **PMBOK** | 项目管理 | 过程组：启动→规划→执行→监控→收尾 |
| **SAFe** | 敏捷规模化 | 需求层次：Epic→Feature→Story |
| **DevOps** | 工程实践 | 流水线：Plan→Code→Build→Test→Release→Operate |

### 2.2 三个核心视角

本体系从三个视角进行组织和管理：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│    【工程视角】              【管理视角】              【资产视角】      │
│    Engineering View         Governance View           Asset View       │
│    ─────────────            ─────────────             ──────────       │
│    "做什么、怎么做"          "管什么、控什么"          "存什么、用什么"   │
│                                                                         │
│    技术活动与产出            决策与质量把控            知识沉淀与复用     │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

**视角说明：**

| 视角 | 关注点 | 典型活动 | 产出 |
|------|--------|---------|------|
| 工程视角 | 技术实现 | 需求分析、架构设计、编码、测试 | 成果物（Artifact） |
| 管理视角 | 质量控制 | 评审、审批、监控、决策 | 评审记录、决策文档 |
| 资产视角 | 知识复用 | 沉淀、标准化、检索、复用 | 数字化资产（Asset） |

## 三、术语体系

### 3.1 核心术语定义

| 概念 | 术语 | 英文 | 定义 |
|------|------|------|------|
| 时间维度 | **阶段** | Phase | 生命周期的时间切分，如"需求阶段"、"开发阶段" |
| 活动类型 | **过程** | Process | 特定目标的系列活动，如"需求定义过程"、"架构设计过程" |
| 工作产出 | **成果物** | Artifact | 过程产出的具体内容（文档、代码、配置等） |
| 沉淀价值 | **资产** | Asset | 可复用的结构化成果物 |
| 质量关卡 | **评审点** | Review Gate | 阶段间的质量门禁 |
| 关联关系 | **追溯** | Traceability | 成果物之间的关联关系 |
| 转化过程 | **资产化** | Assetization | 成果物转化为资产的过程 |

### 3.2 术语使用规范

```
示例用法：
- "在【需求阶段】，执行【需求定义过程】，产出【需求规格说明书】成果物"
- "成果物经过评审后，完成资产化，成为可复用的【数字化资产】"
- "通过【追溯机制】，可以追踪每个API设计对应的业务场景"
```

### 3.3 缩写对照

| 缩写 | 全称 | 含义 |
|------|------|------|
| SLDC | Software Lifecycle Development | 软件生命周期开发 |
| BPMN | Business Process Model and Notation | 业务流程模型与符号 |
| DDD | Domain-Driven Design | 领域驱动设计 |
| SOP | Standard Operating Procedure | 标准操作流程 |
| KPI | Key Performance Indicator | 关键绩效指标 |
| CI/CD | Continuous Integration/Deployment | 持续集成/部署 |
